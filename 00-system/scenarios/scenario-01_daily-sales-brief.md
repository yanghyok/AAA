## 상황 설정

**월요일 아침 9시 47분. 청담 쇼룸 2층 사무실.**

창밖으로 선릉로 가로수가 5월 햇빛에 반짝인다. 어제는 일요일이라 매출 데이터를 못 봤다. 자사몰 가정의 달 프로모션 마지막 주차였고, 오늘의집 정기 큐레이션도 같이 걸려있었던 주차다.

플로우 알림이 울린다.

> **김지연님**: 소연님, 10시 30분에 지난주 매출 정리 같이 보는 자리 잡혔어요
>
> 가정의 달 프로모션 마지막 주차였잖아요
> 오늘의집 큐레이션도 같이 걸려있어서 두 개 효과 분리해서 보면 좋을 것 같아요

**(속으로)**: 한 시간도 안 남았네. 지난주 7일치 채널별·시리즈별 매출 + 광고비 + 오프라인 쇼룸까지 다 보는데, 평소 같으면 엑셀 4개 열고 VLOOKUP 두어 번 돌리고, 2023~2026년 합쳐놓은 매출 파일이 너무 무거워서 열리는 데만 3분.

플로우 알림이 또 온다.

> **김은수님**: 소연 부서장님, 일요일에 자사몰 라이프스타일 카테고리 주문이 갑자기 늘었던데 혹시 보셨어요? 발주 판단할 때 같이 보고 싶어서요

**(속으로)**: 모두가 같은 데이터를 본인 각도로 본다... 그런데 데이터는 한 곳에 모여있지 않다. 자사몰 카페24 관리자 페이지 + 29CM 셀러센터 + 오늘의집 관리자 + 네이버 스마트스토어 + 쿠팡 윙 + 청담/강남 쇼룸 수기 매출 파일.

- 진짜 마감: 10:30 (매출 리뷰 미팅)
- 체감 마감: 10:20 (회의실 이동 시간)
- 실제 가용 시간: 약 30분

**Claude Code가 없었다면?**
- 5개 입점몰 관리자 페이지 각각 로그인 → 전주 매출 CSV 다운로드 (40분)
- 자사몰 카페24에서 전주 데이터 export (15분)
- 청담·강남 쇼룸 수기 매출 파일 받기 + 형식 맞추기 (20분)
- 채널별 합계, 시리즈별 합계, 전주 대비 변화율 계산 (30분)
- 광고비 매출 매칭, ROAS 계산 (20분)
- 미팅 보고용 한 장 정리 (15분)
- 총 **2시간 20분, 미팅 못 가고 패닉**

**Claude Code가 있다면?**
- 분석 요청 (5분)

- 김은수님 발주 판단용 자료 따로 정리 (5분)
- 총 **25분, 회의실 가기 전 커피 한 잔**

## 미션 1: 지난주 채널·시리즈별 매출 + 변화 원인

미팅에서 가장 먼저 나올 질문. "가정의 달 프로모션이랑 오늘의집 큐레이션 효과가 분리됐는지." 두 이벤트가 같은 주차에 겹쳐 있어서, 어디까지가 무엇 덕분인지 끊어내는 게 핵심이다.

> **김지연님**: 미팅에서 제일 먼저 나올 포인트는
> "두 개 효과가 어디서 분리되는지"일 거예요
>
> 단순히 "지난주 매출 X.X백만원, 전주 대비 +X%"로 끝나면 안 되고
> "프로모션이 어디까지 끌어올렸고, 오늘의집 큐레이션이 어디서 더 붙었는지"가 같이 보였으면 해요

**(속으로)**: 두 개 효과 분리... 채널별로 떼서 보면 자사몰은 프로모션, 오늘의집은 큐레이션이니까 변화율이 다르게 나올 거다. 시리즈별로도 떼서 봐야 한다.

**분석 요청**:
```
@sample-data/rareraw/online_sales.csv
@sample-data/rareraw/offline_sales.csv
@sample-data/rareraw/products_master.csv

지난주 (2026-05-11 ~ 2026-05-17) 레어로우 매출을 분석해줘.

[분석 프레임]
1) 시간 비교축
   - 지난주 7일 vs 직전 7일 (2026-05-04 ~ 2026-05-10) 변화율
   - 단순 합계가 아니라 "왜 움직였는가"를 같이 설명

2) 분리축
   - 채널별: 자사몰 · 29CM · 오늘의집 · 네이버 · 쿠팡 · 청담쇼룸 · 강남쇼룸
   - 시리즈별: 랙 · 시스템000 · 워크 · 홈 · 라이프스타일

3) 효과 분리 (핵심)
   - 가정의 달 프로모션(5/1~5/10): 자사몰 중심에서 어디까지 끌어올렸나
   - 오늘의집 큐레이션(5/5~5/8): 오늘의집 채널에서 어디까지 추가로 붙었나
   - 두 이벤트가 겹친 채널·시리즈는 따로 표시

4) 출력 흐름
   - 표 1: 채널별 어제 매출 / 7일 평균 / 변화율 / 한 줄 비고
   - 표 2: 시리즈별 동일 구조
   - 변화 원인 카드: 변화율 ±15% 이상 채널, ±20% 이상 시리즈마다 한 장씩
     "[+34.5%] 채널/시리즈 — 추정 원인(프로모션·큐레이션·시즌·데이터 누락 중 하나) — 다음 액션 한 줄"
   - 마무리 한 단락(200자 이내): 메신저로 공유할 수 있는 요약
```

**결과 확인 후 추가 질문 예시**:
```
"보고용으로 한 장에 압축해줘. 숫자 나열 말고
'무엇이 일어났는가 → 왜 일어났는가 → 다음 액션은 무엇인가' 흐름으로."
```

## 미션 2: 김은수님 발주 판단용 — 라이프스타일 카테고리 급증 원인

매출 리뷰 미팅 끝나고, 김은수님 요청을 처리한다.

> **김은수님**: 일요일에 라이프스타일 카테고리 주문이 갑자기 늘어서요
> 단순 자연 변동인지, 자사몰 컬러 런칭(4/20~4/26) 후속 효과가 늦게 들어온 건지
> 발주 판단할 때 참고하려고요
>
> 어제 자사몰만 늘었으면 컬러 런칭 후속이고, 입점몰도 다 늘었으면 자연 변동일 거예요

**(속으로)**: 김은수님은 발주 판단할 때 등급별 안전재고 기준으로 보니까, 채널별로 분리된 데이터 + SKU 단위로 떨어진 표가 필요하다.

**분석 요청**:
```
@sample-data/rareraw/online_sales.csv
@sample-data/rareraw/products_master.csv

라이프스타일 시리즈(10 COLORS) SKU의 어제(2026-05-17) 매출이
직전 7일 평균 대비 +50% 이상 늘어난 SKU만 골라줘.

[분류 프레임]
- 자사몰에서만 늘었으면 → "컬러 런칭 후속"
- 입점몰에서도 함께 늘었으면 → "자연 변동 (시즌·주말)"
- 한 채널만 비정상적으로 늘었으면 → "데이터 확인 필요"

[출력]
SKU별로 한 줄씩:
[SKU명] | 어제 매출 | 7일 평균 | 변화율 | 자사몰/입점몰 분리 변화율 | 분류 | 권장 다음 행동
```

**결과 확인 후 추가 질문 예시**:
```
"김은수님 발주 판단용으로 SKU 옆에 권장 발주 수량 계산해줘.
최근 한 달 판매 추세 + 직전 14일 일평균 판매량 기준으로."
```
