# 프로젝트·계약 트래커 브리프 — 분석 템플릿

> 여러 프로젝트·계약 조건·미팅 액션 아이템이 흩어져 있을 때, 한 화면에 모아 "지금 가장 우려되는 한 곳"을 짚기 위한 양식. 위클리 부사장 보고·법무팀 정기 점검·디자인 회사 PM 회의·HR 인력현황 검토 모두 동일 패턴.

---

## 입력

- `projects_master.csv` — 프로젝트 마스터 (작품·사업·인사 단위 모두 호환)
- `contract_terms.csv` — 작품별·거래처별 계약 조건 (권리·정산·홀드백·승인)
- `meeting_actions.csv` — 위클리 미팅에서 나온 액션 아이템 (owner·due_date·severity·status)

---

## 출력 구조

A4 1장 또는 한 화면. 다음 3개 섹션.

### 섹션 1. 헤드라인 (오늘 가장 우려되는 한 곳)
- 1줄: "어느 프로젝트의 어떤 상태가 어떤 마감을 위협하는가".
- 근거 액션 ID 1~2개 인용.

### 섹션 2. 프로젝트 상태 점검표
- 프로젝트 × 상태 × 우선순위 × 다음 마감.
- 컬러 알림:
  - 🔴 지연 (status=지연 또는 due_date 초과)
  - 🟡 임박 (due_date ≤ 7일)
  - 🟢 정상
- 같은 프로젝트에 액션이 3건 이상 쌓여 있으면 별도 표시.

### 섹션 3. 계약 조건 비교 카드
- 작품별 권리·정산·홀드백·승인을 한 표에 모음.
- "표준화 어려운 지점" 강조: 정산 방식이 비율(60:40) vs 로열티(%) 혼재, 홀드백 길이 편차, 승인 요건 유무.
- 진행 중 협상 건은 별도 표시 + 후속 액션 ID 인용.

---

## 검출 규칙

| 신호 | 임계값 |
|------|--------|
| **지연 위험** | status=지연, 또는 status=진행중인데 due_date - 오늘 ≤ 3일 |
| **고심각도 + 미완료** | severity=상이면서 status가 완료가 아닌 건 |
| **액션 누적** | 같은 project_id에 미완료 액션 3건 이상 |
| **계약 미체결 + 마감 임박** | contract.status ≠ 체결완료이고, project.premiere_or_deliv까지 60일 이하 |
| **승인 의존 위험** | approval_required=Y인 프로젝트 중 임원·외부 승인 대기 액션이 있는 경우 |

---

## 길이 제약

- **A4 1장**: 헤드라인 → 점검표 → 계약 비교. 표는 8행 이하 가독성 우선.
- **헤드라인**: 1줄 50자 이하.
- **상태 점검표 한 행**: 5칸 (project / status / priority / next_due / 🔴🟡🟢).
- **계약 비교 카드**: 6칸 (project / 권리 / 정산 / 홀드백 / 승인 / 후속).

---

## 톤 가이드

- **부사장 위클리 보고**: 헤드라인 1줄 + 점검표 + 우려 사유 2~3줄. 계약 카드는 별첨.
- **법무팀 내부 점검**: 계약 카드 + 협상 중 건 후속 액션 분명히.
- **PM 회의 자료**: 점검표 + 액션 아이템 owner별 묶음.

---

## Open Questions

- 같은 프로젝트에서 미팅이 여러 번 열렸을 때, 액션 아이템 중복(같은 작업이 회차마다 등장)을 어떻게 묶을지 → 같은 owner·같은 키워드면 묶고 회차 표시.
- 계약 표준화가 어려운 지점을 "데이터 차이"로 보여줄지 "이슈 카드"로 따로 뽑을지 → 비교 카드의 차이가 3건 이상이면 별도 이슈 카드.
- 승인 의존 액션은 owner를 외부로 둘지 내부로 둘지 → "외부 승인" 칼럼 추가 검토.
